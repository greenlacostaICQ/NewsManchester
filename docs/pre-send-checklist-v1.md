# Pre-Send Checklist v1

Короткий практический чек перед отправкой ежедневного выпуска.

## 1. Широкий скан

Перед сборкой проверены все обязательные слои:

- media-layer: `MEN`, `BBC Manchester`, `ManchesterWorld`, `ITV Granada`, `The Mill`, `The Manc`
- transport: `TfGM`, `National Rail`, relevant council roads pages
- police / incidents: `GMP`
- public services: `NHS / trust / ITV`, если есть активный сбой или забастовка
- culture weekly: минимум `1` theatre source + `1` museum / exhibition source + `1` major what's-on source
- venues / tickets: official pages `Co-op Live`, `AO Arena`, `Factory International` и другие релевантные official event pages
- food / openings: минимум `1` сильный source
- football: `MUFC`, `MCFC`, `Salford City`

## 2. Межвыпускной dedupe

- Каждый кандидат проверен против вчерашнего выпуска.
- Без новой фазы факт удалён.
- Carry-over разрешён только если:
  - `Актуально сегодня`
  - `Продолжается`
  - `Сегодня вступает в силу`
- Carry-over без label запрещён.

## 3. Внутривыпускной dedupe

- Один смысловой факт живёт в одном основном блоке.
- `Коротко` может только коротко сослаться на факт, но не пересказывать его целиком.
- `Радар по районам` выключен, если он просто пересказывает верхние блоки.
- `NHS / public services` не создан отдельным блоком, если его единственный факт уже раскрыт выше.

## 4. Самодостаточность текста

- Первая фраза каждого пункта понятна без чтения вчерашнего выпуска.
- Нет формулировок:
  - `после инцидента`
  - `после нападения`
  - `по этой истории`
  - `по этому делу`
  - если перед этим не назван сам факт
- У каждого нетривиального названия есть пояснение:
  - артист -> жанр
  - спектакль -> жанр / тип постановки
  - выставка -> тема
  - компания -> сектор
  - термин полиции / суда -> человеческая расшифровка

## 5. Источники и ссылки

- Для события используется official event page, если она существует.
- Не использовать:
  - homepage venue вместо event page
  - `/amp/`
  - search pages
  - reseller links как primary
- Source labels в тексте человекочитаемы.

## 6. Блоки

Обязательные:

- шапка
- Коротко
- Транспорт и сбои
- Что важно сегодня
- За последние 24 часа
- Что важно в ближайшие 7 дней
- Билеты / Ticket Radar

Условные:

- Дальние анонсы
- Открытия и еда
- IT и бизнес
- Радар по районам

Правило:

- пустые low-signal блоки скрывать полностью
- не печатать:
  - `Новых открытий не добавляю`
  - `Новых сделок не добавляю`
  - `Районных сигналов нет`

## 7. Public services

- Если активная забастовка / disruption уже идёт сегодня, факт нельзя просто выбросить как `вчерашний`.
- Он должен:
  - либо попасть в выпуск с практическим углом на сегодня
  - либо быть явно исключён в debug output с причиной

## 8. Финальный red-team pass

Перед отправкой задать к каждому пункту 4 вопроса:

- что случилось
- где это
- когда это
- на что это влияет мне сегодня / на неделе

Если хотя бы на один вопрос нет ответа, пункт переписывается или удаляется.
