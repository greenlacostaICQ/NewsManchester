# Agent Writer — инструкция для agentic rewrite

Этот документ описывает шаг, который выполняет **агент** (Codex / Antigravity / любой LLM-агент с доступом к файловой системе) между Phase 1 (collect/dedupe/validate) и Phase 2 (write/edit/build/send) пайплайна.

## Контекст

После Phase 1 в файле `data/state/candidates.json` лежат все кандидаты с заполненными полями:
- `title` — заголовок из источника (английский)
- `summary` — описание/lead из источника (английский)
- `practical_angle` — автоматически сгенерированный практический угол (русский)
- `lead` — ведущее предложение (английский)
- `primary_block` — в какой блок дайджеста идёт этот кандидат
- `source_label` — короткое имя источника
- `include` — true/false, включать ли в выпуск
- `validation_errors` — список ошибок валидации (пустой = OK)

Поле `draft_line` у большинства кандидатов **пустое или механическое**. Задача агента — написать для каждого подходящего кандидата качественный `draft_line` на русском языке.

## Что делать

### 1. Прочитать файл
```
data/state/candidates.json
```

### 2. Для каждого кандидата, где `include: true` И `validation_errors` пуст:

Написать поле `draft_line` — **один готовый bullet** (пункт дайджеста) на русском языке.

#### Формат draft_line
```
• Лид-фраза. Конкретика (что/где/когда). Практический угол (why it matters / что делать). Source label
```

#### Правила (из editorial-style-guide-v1.1.md)

**Обязательно:**
- Язык: **русский**. Названия клубов, площадок, компаний, районов — в оригинале на английском.
- Каждый пункт отвечает на: **что произошло, где, когда, почему это важно, что делать**.
- Первая фраза — самодостаточная: читатель понимает пункт без знания вчерашнего выпуска.
- При первом упоминании: артист → с жанром, событие → с форматом, место → с районом, компания → с пояснением.
- `source_label` всегда в конце как подпись.
- Обычный item: 2–4 предложения. Главный item: 4–6 предложений.

**Запрещено:**
- Изобретать факты, ссылки, даты, цены, которых нет в `title`/`summary`/`lead`.
- Добавлять детали, которые не содержатся в полях кандидата (`title`, `summary`, `lead`, `practical_angle`) — даже если эти детали можно проверить внешне. Названия альбомов, промокоды, дополнительные маршруты, контекст из открытых источников — всё это запрещено добавлять в `draft_line`, если этого нет в самом кандидате. Кандидат — единственный источник для этого шага.
- Писать от первого лица: «я не вижу», «я не нашёл», «у меня нет подтверждения».
- Использовать мета-комментарии: «это не просто X», «это заметное», «как фон недели».
- Использовать англо-русскую мешанину: «civic signal», «дорожный сюжет», «новая волна лайнапа».
- Закрывать пункт расплывчато: «это важный сигнал», «стоит знать», «это новый маршрут» — вместо конкретного глагола действия для читателя.
- Подменять смысл исходного факта.

**По блокам:**
- `weather` — минимум/максимум °C, вероятность осадков, один практический вывод.
- `transport` — линия/участок, окно, затронутые станции, workaround. Человеческим языком. Если затронуто несколько маршрутов, называй только те, что прямо влияют на читателя сегодня. Не перечисляй полный технический список — дай то, что меняет конкретное решение.
- `short_actions` — формат строго: `После работы:` / `Сегодня:` / `С 09:00:` + одно действие. Без фактов и цифр. Полный факт живёт в другом блоке ниже.
- `today_focus` — самые важные истории дня, 3–4 предложения, ответ на «зачем мне это знать сегодня».
- `last_24h` — pulse города. Обязательно лид + конкретика + практика.
- `next_7_days` — weekly look-ahead: что, где, когда, зачем.
- `ticket_radar` — событие, venue, дата, presale/general sale, цена (если есть), verdict курсивом.
- `football` — счёт, место в таблице, что это значит, следующий матч.
- `openings` — что это, где (район + адрес), концепт, дата, почему выделяется.

### 3. Записать обновлённый файл
Записать `candidates.json` обратно с заполненными `draft_line`. Остальные поля **не менять**.

### 4. Запустить Phase 2
```bash
bash scripts/run_pipeline_publish.sh
```

## Пример

**Вход (кандидат):**
```json
{
  "title": "Man arrested and drugs seized after Challenger warrant in Old Trafford",
  "summary": "On Wednesday 22 April 2026, officers from Trafford Challenger Unit and Trafford Division executed a warrant...",
  "practical_angle": "Сверить с официальным policing update; учесть, как это влияет на район сегодня.",
  "primary_block": "last_24h",
  "source_label": "GMP",
  "include": true,
  "validation_errors": []
}
```

**Выход (draft_line):**
```
• В среду полиция Greater Manchester провела обыск в Old Trafford по ордеру Trafford Challenger Unit: один человек задержан, изъяты наркотики. Если живёте в районе, следите за обновлениями GMP по этому делу. GMP
```

## Graceful degradation

Если агент не запущен или не доступен, `run_daily_digest.sh` по-прежнему работает: `writer.py` соберёт механический `draft_line` из `title + summary + practical_angle`. Текст будет хуже, но pipeline не сломается.
